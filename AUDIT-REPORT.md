## 5. Варіації товарів

### Як це працює в CS-Cart

1. **Товар-батько** (`parent_product_id = 0`) — основний товар
2. **Товар-дитина** (`parent_product_id > 0`) — варіація з конкретним значенням feature

**Приклад:** "Масло для кутикули HELLO Coconut Cookie"
- Варіація по `feature 134 (Вибір об'єму)`: "3 мл"
- `feature_style: dropdown_labels`
- `purpose: group_catalog_item`

### 5 типів варіацій в CS-Cart

| Feature ID | Назва | Приклад |
|-----------|-------|---------|
| 134 | Вибір об'єму | 3 мл / 9 мл / 15 мл / 30 мл |
| 133 | Вибір відтінку | №01, №02, №03... |
| 137 | Колір колекції | Рожевий / Бежевий / Білий |
| 141 | Розмір | S / M / L |
| 139 | Інший варіант | Довільний |

### Вплив на ціну
Варіації мають **окрему ціну** (поле `price` кожного product-варіації).  
Наприклад: 3 мл = 45 грн, 15 мл = 120 грн.

## 10. Статуси замовлень

**Endpoint:** `GET /api/statuses/`

| Код | Назва | Колір | Оплата | Склад |
|-----|-------|-------|--------|-------|
| **O** | Відкритий | 🟠 #ff9522 | Ні | Decline |
| **G** | Передано в 1С | 🟢 #38761d | Ні | Decline |
| **H** | Передати в 1С | 🟡 #ff9900 | Ні | Decline |
| **C** | Зібрано | 🟢 #97cf4d | Так | Decline |
| **P** | Оброблено - Оплачено - Відправлено | 🟢 #97cf4d | Так | Decline |
| **Y** | Не змогли зв'язатися | 🔴 #cc4125 | Ні | Decline |
| **B** | Скасовано - Клієнт відмовився | 🔴 #ff0000 | Ні | Increase |
| **I** | Скасовано - Немає в наявності | 🔴 #ff0000 | Ні | Increase |
| **D** | Відхилено | 🔴 #ff5215 | Ні | Increase |
| **F** | Невдача | 🔴 #ff5215 | Ні | Increase |

### Workflow замовлення
```
O (Відкритий) → G (Передано в 1С) → H (Передати в 1С) → C (Зібрано) → P (Оплачено/Відправлено)
                                                           ↓
                                            Y (Не змогли зв'язатися)
                                            B (Клієнт відмовився)
                                            I (Немає в наявності)
```

```
Authorization: Basic {base64(email:api_key)}
```

### Важливі нюанси
1. **Зображення** — не повертаються в списку `/api/products/`, тільки в деталях `/api/products/{id}`
2. **Features** — правильний endpoint `/api/features/`, а НЕ `/api/product_features/`
3. **Знижки** — `base_price > price` (не list_price!)
4. **Варіації** — через `variation_features` з purpose=`group_catalog_item`, НЕ через `product_options`
5. **B2B ціни** — через usergroup pricing, НЕ через окреме поле
6. **Промо** — endpoint `/api/promotions/` не працює (404)
